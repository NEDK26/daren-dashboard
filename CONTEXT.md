# CONTEXT.md

本项目的领域术语表（Ubiquitous Language）。

---

## 核心实体

### 达人（Daren）

跨平台的 KOL/创作者实体，在同一批次内按 `nickname`（全网统一昵称）唯一标识。一个达人可拥有多个平台的视频数据；同一个昵称可以在不同批次中各有一条达人记录。在 Excel 导入时，通过批次内昵称匹配已有达人，不存在则在该批次新建。

- **属性**：昵称、机构名称、内容类型、达人分类、粉丝数、主页链接、账号、平台昵称
- **关系**：一个达人有多条视频

### 达人分类（Daren Category）

导入 Excel 为达人指定的业务分类，用于在单个批次内组织和筛选达人。筛选项仅取自当前批次实际存在的非空分类，并保持导入时首次出现的顺序。

### 内容类型（Content Type）

导入 Excel 为达人指定的内容形态，用于在单个批次内组织和筛选达人。筛选项仅取自当前批次实际存在的非空内容类型，并保持导入时首次出现的顺序。

### 视频（Video）

达人发布的作品，是系统中最细粒度的数据单位。每条视频关联一个达人和一个平台；同一作品在共创场景下可以形成多条达人关联记录。

- **标识符**：`videos.id`（平台内部主键）；`work_id` 是外部系统提供的作品 ID，可被多个共创达人共享
- **关键属性**：标题、描述标签、内容链接、时长、发布时间、DA/7日播放量/点赞量、评论/收藏/转发量、违规/合规状态、节点/爆款/申诉标记
- **截图**：播放量截图、点赞量截图、7日播放量截图、7日点赞量截图（上传至本地 `uploads/` 目录）
- **异常数据标记（Data Anomaly Marker）**：表示某个视频数据单元格需要核查。标记可由 Excel 红底单元格产生，也可由管理员手动标记或取消；手动操作不修改单元格的业务数据，标记统一纳入异常统计。管理员手动操作的范围仅限四个截图字段：播放截图、点赞截图、`7日播放` 截图、`7日点赞` 截图。

### 批次（Batch）

管理员主动创建的一个数据核对周期，一个批次对应一份 Excel 文件。批次名称固定为 `YYYY年MM月｜ 自定义标题`，年份、月份和自定义标题必填，完整名称全局唯一；系统同时记录原始文件名和导入时间。系统最多保留一个草稿批次、一个已发布批次，历史批次可以有多个；已有草稿时不能创建新批次，必须先完成导入或删除草稿。批次创建和导入成功后都保持草稿，不会立即对用户开放；管理员点击发布后才成为当前已发布批次，之前的已发布批次转为历史批次。新批次导入和发布准备期间，用户继续查看原已发布批次。导入失败只回滚本次达人、视频和用户数据，批次继续保持草稿，管理员可以在同一批次下重新导入。已发布批次可以撤销，撤销后恢复上一个已发布批次，被撤销批次回到草稿。已发布批次和历史批次只读；草稿可以删除。达人表和视频表都直接带 `batch_id`，不额外建立批次与达人的关联表。同一批次内再次导入相同的达人、平台和作品 ID 时覆盖该批次的原记录；不同批次保留独立的数据记录，可分别查看。未出现在某批次 Excel 中的达人不属于该批次，但其历史批次数据和登录账号继续保留。现有数据迁移到 `2026年05月｜ 数据核对` 批次，并将该批次设为当前已发布批次。

### 平台（Platform）

视频发布的平台，当前支持：**抖音**（Douyin）、**快手**（Kuaishou）、**B站**（Bilibili）。

### 用户（User）

系统登录账户。分为两种角色：

- **管理员（admin）**：可查看/编辑所有数据，导入导出，配置权限，查看审计日志
- **普通用户（user）**：可以查看和编辑所有批次中 `nickname` 等于自己 `display_name` 的达人数据，可编辑范围受管理员配置限制；用户账号本身不绑定批次

### 初始随机密码（Initial Random Password）

系统为每个新建的普通用户账户单独生成的首次登录凭据；不同达人账户使用不同的随机码，不再共用固定初始密码。同一达人进入后续批次时复用原登录账号，不重新生成初始密码。

### 批次账号清单（Batch Account List）

以单个批次为范围，列出该批次达人对应的全局登录账号及其凭据发放信息。清单按该批次达人记录的达人分类筛选，不以全局用户表的字段筛选。

### 审计日志（Audit Log）

每次数据修改的记录，包含操作人、表名、记录 ID、字段名、旧值、新值、时间。

### 达人删除（Daren Deletion）

永久移除一个达人、该达人的全部视频、这些视频引用的本地截图文件，以及与达人昵称同名的普通用户账户。历史审计日志保留不变。重新导入同名达人时，视为新的达人和账户。批量删除按整批处理：如果任一目标达人不存在或数据库删除失败，则整批不删除；本地截图文件已不存在不视为失败。

### 达人确认状态（Daren Confirmation Status）

归属于达人，表示其对应普通用户对本人视频数据的最终反馈状态；管理员在达人列表中查看该状态。默认“待确认”；数据无误时用户提交后为“已确认”；有修改时用户保存并确认提交后为“已提交申诉”。任何后续视频字段编辑、截图上传或替换，或编辑后暂不提交，均为“待确认”。

### 核对模块（Reconciliation Modules）

用户登录后的工作台入口。目前包含“本期数据核对”和“本期费用核对”两个模块；数据核对进入现有达人和视频数据页面，费用核对暂未开放，仅展示开发中提示。

### 批次展示（Batch Views）

用户默认进入最新的当前批次，批次切换器只显示自己在该批次有数据的批次；如果当前批次没有该用户，则显示“本期暂无数据”。不同批次不合并到同一张达人或视频表中。

### 批次导出（Batch Export）

导出必须绑定单个批次，默认导出当前批次；历史批次可以单独导出，不合并多个批次。

### 批次统计（Batch Metrics）

达人确认状态统计、视频异常数量和已提交异常数量，均按当前选中的批次计算。

## 展示模式

从两级导航（达人列表 → 按平台筛选视频）改为一级导航（达人列表 → 该达人所有平台的视频）：

- **达人列表页**：展示所有达人及其属性，点击昵称进入视频明细
- **视频明细页**：展示该达人所有平台、所有字段的视频数据，默认全部平台，可按平台/违规/合规/标题筛选，按"平台分组 → 发布时间降序"排列

---

## 权限模型

- 登录态通过 Session（`express-session`）维护，24 小时过期
- 普通用户的数据库访问按 `nickname = display_name` 过滤所有批次的达人记录，再按达人记录关联过滤视频
- `editable_columns` 是全局的普通用户可编辑列配置；所有普通用户只能编辑自己名下视频中被配置的字段
- 普通用户上传或替换截图必须先进入对应视频的编辑状态；管理员不受此界面限制
- 达人删除仅管理员可用
