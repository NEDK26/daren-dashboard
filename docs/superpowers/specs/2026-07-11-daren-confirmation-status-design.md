# 达人确认状态设计

## 目标

让每位普通用户确认本人视频数据，并让管理员在达人列表中看到确认结果。

## 状态

状态归属于达人，数据库值和展示文案均为以下三种：

- `待确认`：默认状态；任何视频字段保存、截图上传或截图替换后重置为此状态。
- `已确认`：用户确认数据无误后提交。
- `已提交申诉`：用户修改数据后，在保存或上传后的确认弹窗中选择提交修改。

管理员只查看状态，不提供处理、驳回或审批动作。用户编辑后选择暂不提交时，数据照常保存，状态保持或回到“待确认”。

## 数据与接口

`darens` 新增 `confirmation_status TEXT NOT NULL DEFAULT '待确认'`，并为已有数据库执行兼容迁移。达人列表查询返回此字段。

新增已登录用户可调用的确认接口 `PUT /api/darens/:id/confirmation`，仅允许普通用户更新与自己 `display_name` 同名的达人；管理员不需要通过此入口变更状态。接口仅接受“已确认”和“已提交申诉”。

视频字段保存与截图上传均在后端将所属达人重置为“待确认”。因此绕过前端的请求也不会保留过期确认状态。

## 用户端交互

普通用户直接进入自己的视频数据页。页面显示当前状态；在“待确认”时提供“确认数据无误”按钮，提交后显示“已确认”。

用户保存至少一个实际变化的视频字段后，先保存数据并重置状态，再显示“是否确认提交修改”弹窗。确认时调用确认接口并显示“已提交申诉”；取消时保留保存的数据和“待确认”状态。截图上传或替换采用相同流程。

管理员保留现有视频编辑体验，不显示普通用户确认按钮或修改确认状态。

## 管理员端交互

达人列表新增“状态”列，显示“待确认”“已确认”或“已提交申诉”。状态仅用于查看，搜索、筛选和审批不在本次范围内。

## 权限与审计

普通用户只能确认和修改自己的达人视频数据。管理员继续拥有现有全量编辑权限。状态变更写入审计日志，便于追溯确认与提交申诉的操作人。

## 验收与测试

- 新建和旧达人均显示“待确认”。
- 普通用户确认无误后，管理员列表显示“已确认”。
- 普通用户保存任何一项视频修改或上传截图后，状态先为“待确认”；确认提交后显示“已提交申诉”。
- 选择暂不确认时，数据已保存且状态为“待确认”。
- 越权用户不能确认其他达人的状态，也不能保留修改前的“已确认”。
