# Excel 导入性能与进度反馈设计

## 根因

当前 sql.js 包装器的每次 `prepare().run()` 都会序列化整个数据库并写入 `data.db`。导入一万行时，视频写入和新增达人/用户写入会触发大量全量落盘。

## 方案

- 增加数据库事务 helper，事务期间禁止逐条 `saveDb()`。
- 导入过程包在一个事务中，全部成功后提交并只落盘一次；发生异常时回滚且不保存中间状态。
- 保留现有导入统计和重复数据处理逻辑，不改变 Excel 格式。

## 前端反馈

- 导入按钮点击后打开不可关闭的处理弹窗，避免用户重复上传或误以为页面卡死。
- 弹窗用 CSS 转圈动画和阶段文案循环展示：上传文件、解析 Excel、批量写入、整理结果。
- 不显示伪造的百分比；请求完成后显示后端返回的真实新增/跳过/新建用户统计。

## 验证

- 测试事务 helper 在成功时提交、失败时回滚。
- 测试导入路由使用事务 helper。
- 测试前端存在导入阶段动画并在请求期间锁定弹窗。
- 运行完整测试和生产构建。
