# ADR 0001：视频明细页从"按平台展示"改为"跨平台统一展示"

- **日期**：2026-07-11
- **状态**：已接受

## 背景

此前达人视频数据采用两级导航：
1. 达人列表页 → 点击平台按钮（抖音/快手/B站）→ 进入该达人+该平台的视频明细

平台按钮仅展示有数据的平台，无数据的灰色禁用。

## 决策

改为一级导航：达人列表页 → 点击昵称 → 进入该达人**所有平台**的视频明细。

在视频明细页内部：
- 新增"平台"列用于区分视频来源
- 默认展示所有平台的视频，保留平台下拉筛选
- 视频排序改为"先按平台分组，再按发布时间降序"

## 影响

### 前端变更
- 达人列表页：移除平台按钮（`platforms` 聚合数据不再需要）、移除 `platform` 和 `is_main_platform` 列、点击昵称跳转视频明细
- 视频明细页：新增平台列、违规描述/合规描述/节点/爆款/申诉/时长等列、新增标题搜索、移除日期筛选、标题改为 `{昵称} — 视频明细`
- 设置页：新增 8 个可编辑列配置项，默认对普通用户关闭

### 后端变更
- `GET /api/darens` 去掉 `GROUP_CONCAT(DISTINCT v.platform)` 聚合
- `GET /api/darens/:id/videos` 不需要改（已支持无 platform 参数查询全部视频）
- `GET /api/export` 去掉 `platform` 筛选参数
- 达人列表不再展示 `platform`、`is_main_platform` 字段

### 不变的
- 权限模型（admin/普通用户数据隔离）
- 审计日志
- 截图上传
- 行内编辑机制
- Excel 导入逻辑

## 备选方案

1. **保持两级导航但优化平台选择 UI**（如 Tab 切换）—— 未采纳，用户希望直接看全部视频
2. **完全去掉达人列表，直接展示全局视频列表** —— 未采纳，达人维度仍有聚合意义
