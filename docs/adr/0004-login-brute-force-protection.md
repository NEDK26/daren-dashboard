# ADR 0004：登录接口采用 IP 与用户名组合限流

## 状态

已接受

## 背景

登录接口当前只校验用户名和密码，没有失败次数限制。攻击者可以持续尝试密码组合，或通过切换用户名继续消耗 bcrypt 和服务器资源。

## 决策

- 不接入验证码。
- 按“客户端 IP + 规范化用户名”维护失败次数。
- 同时维护 IP 级别的失败次数，防止通过切换用户名绕过限制。
- 10 分钟窗口内单个组合失败 5 次后锁定 15 分钟；IP 级别使用更高的聚合阈值。
- 被锁定时返回 HTTP 429 和 `Retry-After`，错误文案不区分账号是否存在。
- 成功登录后清除该 IP 和用户名组合的失败记录。
- 限流状态保存在进程内存中；当前 PM2 配置为单进程。未来扩展多进程或多实例时迁移到 Redis 等共享存储。

## 取舍

内存限流不需要新增依赖和数据库写入，不会把登录攻击记录写入 SQLite；代价是 PM2 重启后计数清空，多实例之间不能共享状态。验证码暂不接入，避免第三方配置和正常用户体验成本。
