# ADR 0006：批次快照与生效规则

批次由管理员主动创建，一个批次只绑定一份 Excel，名称固定为 `YYYY年MM月｜ 自定义标题`，年份、月份和标题必填且完整名称全局唯一；系统最多保留一个草稿批次、一个当前批次，历史批次可以有多个；已有草稿时不能创建新批次。创建后立即保存为草稿。导入成功后才对用户开放并成为当前批次，之前的当前批次转为历史批次；导入期间用户继续查看原当前批次，只有新批次完整导入成功后才切换。导入失败只回滚本次达人、视频和用户数据，批次保留为草稿并允许重新导入；成功批次禁止再次上传，数据有误时必须新建批次。草稿可以删除，当前和历史批次不可删除。达人表和视频表直接保存 `batch_id`，不额外建立批次与达人的关联表；因此同一达人在不同批次中是不同的达人记录，确认状态天然按批次隔离。同一批次内重复导入相同的达人、平台和作品 ID 时覆盖原记录，不同批次保留独立数据快照。用户账号保持全局，通过 `display_name` 匹配所有批次中的同名达人记录。未出现在某批次 Excel 中的达人不属于该批次，但历史数据和账号继续保留。现有数据迁移到 `2026年05月｜ 数据核对` 并设为当前批次。

用户默认进入最新当前批次，批次切换器只展示该用户有数据的批次；当前批次没有该用户时显示“本期暂无数据”，不自动切换到其他批次，不将不同批次的数据合并展示。

草稿和失败批次可以删除；当前和历史批次不可删除，历史批次只读。

导出必须绑定单个批次，默认导出当前批次，历史批次可单独导出，不合并多个批次。

达人确认状态统计、视频异常数量和已提交异常数量均按当前选中的批次计算。
