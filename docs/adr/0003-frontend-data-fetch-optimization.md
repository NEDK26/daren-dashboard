# ADR 0003：列表数据采用服务端分页与可取消筛选请求

## 状态

已接受

## 背景

达人和视频列表当前一次性加载全部记录，表格只在浏览器端分页。达人列表还会为每个达人单独查询异常数据，形成 N+1 查询；视频标题筛选在每次输入变化时都会发起请求。数据量增加后，首屏、筛选和切换达人都会变慢。

## 决策

保留现有 React 18 + Ant Design 结构，做局部优化：

- 达人和视频列表 API 支持 `page`、`pageSize`，返回当前页数据和总数。
- 异常统计改为一次批量查询后在内存中聚合。
- 标题搜索增加短防抖，并取消上一条未完成请求，避免旧响应覆盖新筛选结果。
- 前端只缓存当前视频页的异常字段解析结果。
- 删除保留批量入口，移除达人表格默认逐行删除按钮，降低误删风险。

## 取舍

不引入 React Query、虚拟列表或新的构建系统。当前分页规模和数据量下，服务端分页与请求生命周期管理可以解决主要瓶颈，同时保持部署和维护成本不变。

## 影响

列表 API 的成功响应从数组扩展为 `{ rows, total, page, pageSize }`；前端兼容旧数组响应，避免部署期间前后端短暂不一致导致页面不可用。
